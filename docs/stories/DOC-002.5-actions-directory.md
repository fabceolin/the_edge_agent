### Standard Action Documentation Format

Each action should be documented with:

```markdown
#### `action.name`

Brief description of what the action does.

**Parameters:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `param1` | string | Yes | - | Description |
| `param2` | int | No | 10 | Description |

**Example:**

```yaml
- name: example_usage
  uses: action.name
  with:
    param1: "value"
    param2: 20
  output: result_key
```

**Returns:**

```python
{
    "field1": "value",
    "field2": 123,
    "success": True
}
```

**Notes:**
- Any special considerations
- Error handling behavior
```
