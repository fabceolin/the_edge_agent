## Critical Gaps

### Gap 1 — Story 1.5 has no test design and no trace file (HIGH)
- **Requirement:** Epic Success Criterion 1 ("all 8 child stories merged with passing tests") and DoD bullet 5 ("originating pain points verified").
- **Reality:** `docs/qa/assessments/` contains `TEA-DX-001.5-{nfr,risk}-20260501.md` only. No `-test-design-` and no `-trace-` file. The Story 1.5 source file (`docs/stories/TEA-DX-001.5-dynamic-parallel-error-messages.md`) defines 10 ACs (5 functional, 3 integration, 2 quality).
- **Impact:** 10 of the epic's 109 child-story ACs are **structurally unmapped**. Pain Point 5 (the originating field-report item) has no traced verification path.
- **Severity:** HIGH (matches NFR-AC-6 finding).
- **Action:** Run `*test-design TEA-DX-001.5` followed by `*trace TEA-DX-001.5` before the epic exits Draft.

### Gap 2 — Story 1.3 AC-4 redaction contract unresolved (BLOCKING, security)
- **Requirement:** AC-4 ("reuse the secrets-redaction logic that excludes secrets from checkpoints") + Epic NFR-AC-1.
- **Reality:** No `redact_state` pass exists in the codebase (`checkpoint.py:111` uses raw `pickle.dump`; `cache.py:83 mask_credentials` is text-only). Three Story 1.3 P0 SEC scenarios (UNIT-005, INT-005, INT-006) and two epic-level scenarios (TEA-DX-001-INT-005, INT-006) cannot be authored.
- **Severity:** HIGH — Security FAIL at the NFR layer flows directly into 0% coverage on SEC paths.
- **Action:** Pick one of the three options in `TEA-DX-001.3-nfr-20260501.md` (Option A: build `redact_state`; Option B: no redaction + startup banner; Option C: per-call `sanitize_keys` opt-in). Rewrite AC-4 accordingly. Promote Epic NFR-AC-1 to the epic story body.

### Gap 3 — Story 1.1 AC-4 contradicts `expand_env_vars` semantics (BLOCKING)
- **Requirement:** AC-4 ("missing env vars without a default raise the same error the existing `expand_env_vars` raises elsewhere").
- **Reality:** `expand_env_vars` (`python/src/the_edge_agent/memory/base.py:528-537`) returns `""` for missing un-defaulted vars and **does not raise**. Tests written against AC-4 verbatim will encode the wrong contract. Story DoD forbids modifying `expand_env_vars`.
- **Severity:** HIGH — Story 1.1 cannot be implemented as-written without forking helper semantics.
- **Action:** Rewrite Story 1.1 AC-4 per risk-profile TECH-002 / NFR-AC-1: "missing var → empty string; runner logs WARNING and skips `FileExporter` when `trace_exporter == 'file'` and post-expansion `trace_file` is empty." Story 1.1 trace's UNIT-006/UNIT-007 are already authored against this rewrite.

### Gap 4 — Engine event taxonomy unresolved for Stories 1.3 & 1.7 (BLOCKING for parallel scenarios)
- **Requirement:** 1.3 AC-9, AC-10, AC-11 (parallel/`dynamic_parallel` parent serialization timing); 1.7 timing-mechanism Technical Note.
- **Reality:** `stategraph.py` emits `state` / `parallel_state` / `branch_complete` / `parallel_complete` / `final` / `error`. Neither `node_complete` nor `node_start` exist. The CLI synthesizes equivalents from `state` events at `cli.py:1821-1824`.
- **Severity:** MEDIUM — affects 5 P0 scenarios (1.3 INT-010/011/UNIT-006; 1.7 INT-003/008).
- **Action:** Pin event-mapping decision in 1.3 Technical Notes and 1.7 Technical Notes before P0 test authoring. Promote 1.3 AC-13 from proposed to accepted.

### Gap 5 — Six Epic NFR-ACs not promoted to story body (MEDIUM)
- **Requirement:** Epic-level test contract (NFR-AC-1..6 in NFR doc).
- **Reality:** All six recommendations live in `docs/qa/assessments/TEA-DX-001-nfr-20260501.md` only. The epic story's "Definition of Done (Epic)" section contains overlapping bullets but the NFR-ACs encode the *enforcement mechanism* (snapshot tests, parity tests, file-presence audit) — DoD bullets are advisory.
- **Severity:** MEDIUM — without promotion, the bundle's regression net is not contractually owned by any story PR. Sprint pressure can skip the snapshot tests, the examples-yaml smoke, the cross-story ordering, etc.
- **Action:** Edit the epic story to add the six Epic NFR-ACs as Acceptance Criteria. Each should reference its mechanizing scenario (epic-level INT-001..010).

### Gap 6 — 4 NFR-recommended ACs not adopted in 1.6 (MEDIUM, security regression vector)
- **Requirement:** Story 1.6 AC-13 (import allow-list), AC-14 (sentinel-file exec guard), AC-15 (refactor parity suite), AC-2 wording revision.
- **Reality:** None promoted to story body. AC-4 forbids *runtime side-effects* but not *imports*; without AC-13 + AC-14, a future contributor importing executor code can silently re-introduce executor pathways.
