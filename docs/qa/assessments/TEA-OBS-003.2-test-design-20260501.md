# Test Design: Story TEA-OBS-003.2 — Payload retention policy & cleanup

Date: 2026-05-01
Designer: Quinn (Test Architect)
Mode: YOLO

## Test Strategy Overview

- **Total test scenarios:** 28
- **Unit tests:** 13 (46%)
- **Integration tests:** 14 (50%)
- **E2E tests:** 1 (4%)
- **Priority distribution:** P0: 11, P1: 11, P2: 5, P3: 1

Strategy notes:

- Most behavior is shell-shaped — Typer subcommand → globbing → `os.path.getmtime` → `Path.unlink()`. The natural test surface is **`typer.testing.CliRunner`** for the `tea trace cleanup` command and `caplog` + direct calls into `_configure_from_settings` for the engine-init warning. Both ride on `tmp_path` + `os.utime()` to control mtimes deterministically.
- **"Integration" here means a CliRunner-driven full-command invocation** with a real (temp) filesystem. There is no DB, no network, no LLM. We do not need a spawned subprocess except for one E2E smoke (#28).
- **Risk-driven gating.** The risk profile (CONCERNS) and NFR assessment (CONCERNS) both promote four items to gating ACs that the story does not yet codify: AC-18 (symlink safety), AC-19 (warning substring assertions), AC-10 amendment (zero-match summary), AC-15 sub-bullet (no `2>&1 >/dev/null` in cron). All four have **P0 tests** in this design — the tests are the forcing function. If the implementing engineer ships without the AC additions, these tests will fail by design.
- **Missing-AC pattern:** The story author was asked to add AC-18/AC-19 before merge. This test design assumes they will be added; if they are not, INT-005, INT-006, and UNIT-002/003 below will be the visible failure that prompts the AC update — same load-bearing strategy used in TEA-DX-001.1-test-design-20260501.
- Tests live in a new file `python/tests/test_trace_cleanup.py` (CLI surface) plus additions to `python/tests/test_yaml_engine_observability.py` (warning surface — already exists per the TEA-DX-001 series).
- **Symlink tests are skipped on Windows** via `@pytest.mark.skipif(sys.platform == "win32", reason="symlinks require admin on Windows")`. Symlinks are only meaningful on POSIX targets where TEA is deployed in production cron.
- We avoid platform-flaky tricks: no `os.chmod(0)` on directories owned by root in CI; permission-error tests use a file inside a directory whose mode is set to `0o555` via `tmp_path` (POSIX) or, on Windows, `pytest.skip()` (since chmod is a no-op).
